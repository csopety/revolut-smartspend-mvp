# SmartSpend MVP Project Brief

## What we are building
A presentation-ready MVP of Revolut SmartSpend for a FinTech school assignment.

## User journey
1. User opens SmartSpend.
2. User sees monthly grocery budget and remaining budget.
3. User enters a grocery basket.
4. User selects max travel time and usual store.
5. App compares 4 grocery chains in Budapest II.
6. App recommends the best store.
7. App explains the calculation.
8. App simulates moving savings into a Revolut-style Pocket.

## What matters most
The algorithm is the core of the MVP. The user must understand why the recommended store is best.

## What not to build
- No real bank account connection.
- No real Revolut login.
- No real payment.
- No real retailer API.
- No required Google Maps API.
- No complex database.
- No unnecessary frontend framework.

## Demo success criteria
The evaluator should be able to:
- Run the project easily.
- Enter a basket.
- See ranked stores.
- See savings calculation.
- Understand the recommendation logic.
- Understand which business-plan features are implemented and which are excluded.
